Getting started

  • Collections

    Document collections are added via data sources or entities you add in entity configuration when you connect a data source collections of that source will automatically added to the Collections menu.

    This section could act as database browser for you, and you could also have customized forms, reports, trigger on your data.

  • Configuration

    This section will provide all you need to create and customize a dashboard for managing and analyzing your data and automating your business workflows.

    Automation's are based on a trigger system, to get it working you need to define a condition of for your trigger and then specify an action.

    • Entity

      Every document type that exist in the system are called entity.

    • Data source

      The only supported data source for now is Mongodb.

    • Dashboards

      You can create your own dashboards using configuration section and add chart, form or report widgets to it.

    • Forms

      Serendip have a built-in form builder that supports arrays and objects of fields in your form.

    • Reports

      Serendip created a default set of reports with a quick sampling of your data. in this process type of fields in your data records will be analyzed and then it will create query forms and columns in the report.

Getting started with the rest API

You can access, search and manage your data using serendip rest API.

  • Generate a token

    To access all api routes you need to set an 'Authorization Bearer' token in your header. 

    // REQUEST
curl --request POST \
 --url https://business.serendip.cloud/api/auth/token \
 --header 'Content-Type: application/x-www-form-urlencoded' \
 --data 'username=YOUR_USERNAME_GOES_HERE&password=YOUR_PASSWORD_GOES_HERE'

  • Business list

    As serendip is a multi business/project dashboard you need to send a _business field to some of API's. to get list of all your registered businesses see the following request.
