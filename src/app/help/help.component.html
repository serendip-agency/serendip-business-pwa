راهنمای داشبورد

  • داشبورد اسناد

    داشبورد اسناد به صورت پیش‌فرض از دیتاسورس‌هایی که در نرم‌افزار ثبت کرده‌اید نمونه‌گیری کرده و گزارش‌های خامی را برای شروع سریعتر برای شما ‌می‌سازد.

    این بخش می‌تواند بدون نیاز به هیچ پیکربندی یک داشبورد برای مدیریت دیتای فعلی شما باشد.

  • پیکربندی نرم‌افزار

    با استفاده از این بخش می‌توانید یک داشبورد کامل برای مدیریت و اتوماسیون دیتای خود پیکربندی کنید. این بخش شامل موارد زیر می‌شود:

    • اشیا

      هر سندی که داخل سرندیپ ثبت ‌می‌شود باید به یک شی متصل شود. این اشیا در واقع همان دسته‌های دیتا هستند.

    • دیتا سورس‌ها

      دیتا سورس‌ها همان دیتابیس‌هایی هستند که می‌توانید به سرندیپ متصل کنید تا امکان تحلیل و مدیریت دیتای آن‌ها را داشته باشید.

    • داشبورد‌ها

      داشبورد‌هایی که از منوی اصلی در دسترس هستند از طریق این بخش قابل اضافه کردن یا پیکربندی هستند. هر داشبورد شامل چند تب خواهد بود که هر کدام می‌توانند یک یا چند کامپوننت مانند نمودار، فرم و گزارش را شامل شوند.

    • فرم‌ها

      این بخش برای ساختن و مدیریت فرم‌ها می‌باشد.ساختار فرم‌هایی که در این بخش تعریف می‌شوند به عنوان ورودی کامپوننت فرم‌ساز استفاده خواهند شد.

    • گزارشات

      سرندیپ به صورت پیش‌فرض می‌تواند گزارشات شما را با نمونه‌گیری از اسناد شما بسازد. اما اگر به دنبال سفارشی کردن فیلد‌های گزارش هستید این بخش به کمک شما خواهد آمد.

آشنایی با API سرندیپ

تمامی امکاناتی که در داشبورد سرندیپ وجود دارد و حتی فراتر از آن، از طریق API سرندیپ در دسترس می‌باشد. برخی از این امکانات:

  • سرویس اسناد

    این سرویس امکان ثبت،‌ ویرایش و حذف اسناد را می‌دهد. هر سند یک آبجکت JSON است. رویداد‌های تغییر اطلاعات می‌توانند از طریق وب‌سوکت برای شما ارسال شوند.

  • سرویس تریگر

    جهت خودکارسازی هر فرآیند می‌توان یک یا چند ماشه تعریف کرد. ماشه‌ها تعریفی از شرط‌ها هستند که توسط آن‌ها فرآیند‌ها یا اقداماتی به صورت خودکار انجام خواهند شد.

  • سرویس ذخیره‌سازی فایل

    سرویسی بر پایه stream برای آپلود و دانلود فایل‌ها بر روی بستر سرندیپ.

  • سرویس ایمیل، پیامک و فکس

    سرندیپ مسیرهای متنوعی برای برقراری ارتباط و ایجاد تعامل با مخاتبانتان در اختیار شما می‌گذارد که از طریق این API در دسترس

شروع کار با API

  • دریافت توکن

    برای کار با تمام API ها نیاز دارید یک رشته توکن امنیتی در header درخواست‌ها قرار دهید. این توکن را با درخواست زیر میتوانید دریافت کنید: 

    // REQUEST
curl --request POST \
 --url https://business.serendip.cloud/api/auth/token \
 --header 'Content-Type: application/x-www-form-urlencoded' \
 --data 'username=YOUR_USERNAME_GOES_HERE&password=YOUR_PASSWORD_GOES_HERE'
